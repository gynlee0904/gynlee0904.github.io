# 개발자 포트폴리오 디자인 컨셉

> 전문적 & 신뢰감 있는 개발자 포트폴리오를 위한 디자인 가이드

---

## 1. 컬러 팔레트

다크 네이비 베이스 + 화이트 + 블루 포인트 한 가지 색상만 사용한다.

| 역할 | 색상 | HEX |
|------|------|-----|
| 베이스 배경 | 다크 네이비 | `#0F172A` |
| 서피스 (카드) | 슬레이트 | `#1E293B` |
| 보더 | 미드 슬레이트 | `#334155` |
| 본문 텍스트 | 오프화이트 | `#F8FAFC` |
| 액센트 (포인트) | 블루 | `#378ADD` |

> 라이트 모드를 선호하면 배경 `#FFFFFF`, 텍스트 `#0F172A`, 포인트 `#2563EB` 조합 사용.

---

## 2. 타이포그래피

- **기본 폰트**: `Pretendard` (한/영 혼용 최적), 또는 `Inter`
- **코드용 폰트**: `JetBrains Mono`
- **폰트 크기 스케일**:
  - Hero 제목: `48–64px`, `font-weight: 700`
  - 섹션 제목: `28–32px`, `font-weight: 600`
  - 본문: `16px`, `font-weight: 400`, `line-height: 1.7`
  - 보조 텍스트: `14px`, `color: #94A3B8`

---

## 3. 레이아웃 원칙

- **최대 너비**: `max-width: 1100px`, 가운데 정렬
- **좌우 여백**: `padding: 0 24px` (모바일 기준)
- **섹션 간격**: `padding: 80px 0` 이상으로 충분히
- **그리드**: 프로젝트 카드는 2열 (모바일은 1열)
- **모바일 대응** 필수 — 채용 담당자가 폰으로 보는 경우 많음

---

## 4. 페이지 구조

### 01. Hero
- 이름 + 직군 한 줄 (`Frontend Engineer`, `Full Stack Developer` 등)
- 핵심 가치 한 문장 (예: "사용자 경험과 코드 품질 모두를 중시하는 개발자")
- CTA 버튼 2개: `프로젝트 보기` (스크롤) + `이력서 다운로드`
- 배경: 단색 또는 미세한 그리드/도트 패턴 (과한 애니메이션 금지)

### 02. Skills
- 카테고리별 태그 그룹으로 표시
  - Frontend: React, TypeScript, Next.js …
  - Backend: Node.js, Python, PostgreSQL …
  - DevOps: Docker, GitHub Actions, Vercel …
- 숙련도 퍼센트 막대(%) 사용 금지 — 신뢰도 떨어짐

### 03. Projects (핵심)
- **3~4개만** 엄선 (많을수록 희석됨)
- 각 카드 구성:
  - 프로젝트명 + 한 줄 설명
  - 사용 기술 스택 태그
  - 나의 역할 (기여도 명시)
  - **결과 수치** (예: "응답 속도 40% 개선", "MAU 2,000명")
  - GitHub 링크 + 라이브 데모 링크
- 썸네일 이미지 또는 스크린샷 포함 권장

### 04. About
- 3~4문장의 짧은 소개
- 개발 철학, 관심 분야, 협업 방식 언급
- 너무 딱딱하지 않게 — 인간적인 면모도 중요

### 05. Contact
- 이메일 + GitHub + LinkedIn 아이콘 링크
- 명확한 CTA 버튼 하나: `이메일 보내기`
- 폼(Form) 보다 직접 링크가 신뢰감 높음

---

## 5. 디테일 & 인터랙션

### 사용하면 좋은 것
- 스크롤 시 `fade-in` 애니메이션 (subtle하게)
- 코드 스니펫 하이라이터 (프로젝트 설명에 활용)
- 다크모드 지원
- 로딩 속도 최적화 (이미지 lazy load, WebP 포맷)
- 커스텀 커서 (선택 사항, 과하지 않게)

### 사용하지 말 것
- 배경 파티클 / 글리치 효과
- 배경 동영상
- 숙련도 퍼센트 막대
- 자동 재생 음악/사운드
- 과한 3D 효과 (Three.js 남용)
